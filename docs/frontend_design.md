前端设计文档（概览）

目标
- 设计一个商用级的前台 POS 与实时桌态界面，支持中文/英文、多设备（平板/桌面）及离线容灾。

技术选型
- 框架：React + Vite（快速开发、PWA 支持）
- 状态管理：简易场景使用 React Context；后期可扩展至 Redux / Zustand
- 通信：WebSocket 实时推送 + REST API
- 本地缓存：IndexedDB / localStorage（断网时缓存操作并回补）
- 样式：CSS Modules / Tailwind 可选（此原型使用简单 CSS）

主要页面与组件
1. 登录 / 权限
- 登录（用户名/密码），角色区分（管理员/收银/服务员）

2. 桌位地图（Dashboard）
- 可视化桌位布局：每桌显示编号、状态（空闲/占用/清理/预订）、已用时/剩余时
- 操作：手动开台、续时、结账、挂单、标记清理
- 实时颜色提示与音/视觉告警（异常占用/超时）

3. POS 结算页（Checkout）
- 当前订单明细、计费策略（分钟计费/包时/折扣/会员价）、支付方式选择（现金/扫码/会员余额）
- 生成收据、打印与开电子发票

4. 订单/会话历史
- 查询历史订单、按日期/营业员/桌号筛选、导出 CSV

5. 设备监控与日志（管理端）
- 台灯状态、网关连通性、未回补事件数、审计日志

交互与 UX 要点
- 桌位操作必须确认（避免误结账）
- 支持快速开台与一键续时，收银流程 <=3 步完成（选择桌位→确认费用→记录支付）
- 多语言（中/英）切换按钮，文本资源可提取为 i18n JSON 文件
- 响应式设计：平板横屏为首选布局

离线与回补策略
- 所有关键操作生成本地事务日志（含时间、操作者、操作类型），在网络恢复时逐条回传并校验幂等性

样式与品牌
- 配色：主色调建议使用深蓝或墨绿（专业、沉稳），辅色用于状态提示（绿色=空闲，红色=占用，橙色=警示）
- 字体：支持中英混排（系统字体即可）
